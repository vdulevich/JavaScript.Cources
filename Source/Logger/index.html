1. As a developer I want to have an ability to log information
Functionality provided by function logger.log(data [, outputType]) in logger.js
2. As a developer I want to have an ability to send log information to the console
Functionality implemented in consoleWriter.js class
3. As a developer I want to have an ability to send log information to an alert window.
Functionality implemented in alertWriter.js class
4. As a developer I want to have an ability to send log information to the current window
Functionality implemented in windowWriter.js class
5. As a developer I want to have an ability to send log information to abstract Web API endpoint.
5.1 Not implemented :(
6. As a developer I want to have an ability to select from built-in logging methods.
Functionality provided by logger.log(data [, outputType]).
Parameter 'outputType' to output data with specific writer. If outputType parameter is ommited, defaut outputType will be used.
Functionality provided by logger.setWriter(name).
Parameter 'name' to use specific writer. If name parameter is ommited or has incorrect value writer will not be setted.
7. As a developer I want to be able to extend the library with custom logging methods.
Functionality provided by loggerExt.registerWriter(writer).
Parameter 'writer' must implement baseWriter class.
8. As a developer I want to library to automatically log all unhandled client-side exceptions
Functionality provided by loggerExt.monitorExceptions().
9. As a developer I want to have an ability to extend library with my own instrumentation methods so that the library can monitor not only exceptions but other events too
Functionality provided by extendet library loggerExt.monitorEventAll(domElement[, unsubscribe]).
Parameter 'domElement' dom element to monitor.
Parameter 'unsubscribe' unsubscribe from logging.
10. As a developer extending the library with my own logging or instrumentation methods I want to have established contracts in a form of base class of API documentation so that I can easily know how to extend the library.
Not sure that it's exactly what is needed but there are a contracts in my library like checks for input params types, params undefined and etc.
11. As a developer extending the library I want to see clear errors and warning in case if I violate the contract of logging or instrumentation methods.
For example abstract class 'baseWriter' check if its instance creating and throw error - throw new Error("Can't create abstract class").
12. As a developer I want an ability to access the library through global variable, AMD module or CommonJS module.
AMD architecture implemented. In app.js loggers assigned to global variables
